 ### 简要解析：什么是**零代码**（结合这个营销中台场景）
零代码，就是**不用写代码、不用后端开发介入**，通过**可视化拖拽、配置参数、选择模板**，就能快速搭建/上线营销活动的模式。

放到这个项目里直白说：
1. **传统模式**：做一场秒杀/抽奖，要后端写接口、前端开发、联调测试，开发周期几十天，烟囱式重复开发，上线慢、改起来麻烦；
2. **零代码模式**：运营/商家直接在中台后台，选秒杀、拼团、优惠券模板，填活动时间、规则、库存、门槛，点发布就上线；
3. 中台提前把**秒杀、集赞、抽奖、拼团、优惠券**这些通用能力封装成标准化组件，用户只做配置，不写代码。

### 对应项目定位的核心价值
用零代码中台解决4个痛点：
- 上线慢：不用开发，**活动从几十天缩短到几天甚至当天上线**
- 峰值崩溃：中台统一做了高并发、缓存、削峰、分布式存储
- 风控弱：内置统一风控、防羊毛党
- 数据不一致：统一数据存储、实时监控，多渠道数据统一可看

简单总结：**零代码=把重复技术能力做成可配置模板，让业务人员直接用，解放开发，快速落地营销活动**。

### 项目定位简要解析

企业级实时零代码活动中台（企业服务黄金赛道）
一、市场背景与行业痛点
2026年，企业营销已进入"实时化、个性化、场景化"时代，但传统营销活动系统存在以下致命问题：
- 无法应对秒杀、抽奖、集赞等活动的峰值流量，经常出现系统崩溃
- 活动开发周期长，无法快速响应市场变化
- 缺乏统一的用户数据管理，无法实现精准营销
- 活动效果无法实时追踪，难以优化营销策略
据IDC数据，2026年中国企业级营销技术市场规模将突破800亿元，其中实时营销中台占比超过30%，是企业数字化营销的核心工具。

二、业务理解与核心价值
实时营销活动中台是企业开展所有线上营销活动的统一平台，通过提供标准化的活动组件与工具，帮助企业快速创建、发布、管理营销活动，同时提升活动效果与ROI。 

### 项目经历要点：
项目定位：面向中小企业、电商、品牌方的零代码实时活动中台，标准化提供秒杀、集赞、抽奖、拼团、优惠券等营销活动 能力，解决传统营销系统峰值崩溃、上线慢、风控弱、数据不一致问题，支撑多渠道活动快速落地，实现活动效果数据实时 可观测，与核心业务落地 
- 基于 SpringBoot3 + MyBatis-Plus 构建中台核心业务，集成 Lombok、Hutool 简化开发，Knife4j 生成开放 API 文档； 实现全局 CORS 跨域、Jackson 序列化 Long 精度丢失解决方案；设计活动、优惠券、用户行为、订单等核心表结构， 联合索引优化查询，支撑多租户营销数据隔离。 
- 多级缓存 + 热点治理：复用Caffeine本地缓存 + Redis Hash分布式缓存两级架构，扩展适配秒杀商品、优惠券、 活动热点数据；引入 HeavyKeeper 热点探测算法识别爆款活动、爆款优惠券热点流量，TTL 管控缓存生命周期，避免 缓存击穿 / 雪崩，接口平均响应时间 降低 62%，缓存命中率稳定在 94% 以上； 
- 削峰与异步批量处理：集成 Apache Pulsar 消息队列，将秒杀下单、集赞行为、抽奖记录、消息通知、数据统计 化批量处理，批量消费降低数据库 IO 压力，系统整体吞吐量提升 4.3 倍，单机峰值 QPS 稳定 
- 分布式存储扩容：从单机 MySQL 迁移至 5000+； 异步 TiDB 分布式数据库，实现活动数据、订单数据、用户行为数据自动分片与水平 扩容，规避分库分表复杂度，保障分布式事务一致性，支撑百万级营销数据持久化； 
- 流量管控与高可用降级：复用 Redis 主从 + 哨兵高可用架构，设计多级降级策略；缓存服务不可用时自动扩容本地缓存  / 直查 TiDB；流量超阈值自动关闭非核心营销功能，保障核心秒杀、优惠券能力可用


### 具体问题：
1. **功能设计**：点赞系统需要哪些基础功能
2. **库表设计**：数据库表结构如何设计
3. **基础并发问题**：如何防止用户重复点赞
4. **数据安全**：分布式下如何保证点赞数据一致性
5. **存储优化**：海量点赞数据如何节约存储空间
6. **查询高并发**：1万用户同时查点赞状态，如何抗压
7. **写入高并发**：1万用户同时点赞，系统如何承载
8. **热点/攻击场景**：爆款/刷接口，10万次点赞如何处理
9. **极端流量场景**：百万用户同时点赞，如何架构兜底
10. **监控告警**：如何实时感知系统压力、故障快速告警
11. **故障降级**：系统出问题时，如何降低对用户的影响
