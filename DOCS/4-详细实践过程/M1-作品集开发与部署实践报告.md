## 三、Nginx部署与安全配置

### 3.1 部署架构

```
Internet → 阿里云安全组(80/443/22) → Docker Nginx(:80)
                                          ├── /           → 作品集静态文件
                                          ├── /api/       → thumb-backend:9199
                                          ├── /doc.html   → Knife4j API文档
                                          ├── /grafana/   → Grafana监控面板
                                          └── /prometheus/ → Prometheus指标
```

### 3.2 Nginx配置要点

```nginx
# 反向代理
upstream backend { server thumb-backend:9199; }
upstream grafana { server thumb-grafana:3000; }
upstream prometheus { server thumb-prometheus:9090; }

# 安全头
add_header X-Frame-Options "DENY" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;

# SPA路由支持
location / {
    root /usr/share/nginx/portfolio;
    try_files $uri $uri/ /index.html;
}

# API代理（带路径重写）
location /api/ { proxy_pass http://backend/; }
location /doc.html { proxy_pass http://backend/api/doc.html; }
location /grafana/ { proxy_pass http://grafana/; }
```

**Docker Compose端口绑定**（仅80对外）：

## 六、问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| TS2307: Cannot find module '@/...' | tsconfig缺少路径别名 | 添加baseUrl + paths配置 |
| TS5101: Option 'baseUrl' is deprecated | TypeScript 7.0变更 | 添加ignoreDeprecations: "6.0" |
| Nginx启动失败(80端口占用) | Docker thumb-nginx已占用80 | 整合到Docker Nginx配置 |
| Knife4j 404 | context-path=/api导致路径偏移 | proxy_pass添加/api前缀 |
| Prometheus启动失败 | prometheus.yml被创建为目录 | 删除目录，重新上传文件 |
| 内部服务端口暴露 | Docker端口默认0.0.0.0绑定 | 改为127.0.0.1:端口:端口 |
