## 五、网络安全全栈设计

### 5.1 安全威胁模型

| 威胁 | 风险等级 | 影响 | 防护措施 |
|------|----------|------|----------|
| DDoS攻击 | 高 | 服务不可用 | Nginx限流+阿里云安全组 |
| API未授权访问 | 高 | 数据泄露 | JWT鉴权+CORS白名单 |
| XSS攻击 | 中 | 脚本注入 | CSP头+输入过滤 |
| CSRF攻击 | 中 | 伪造请求 | SameSite Cookie+Token |
| 中间人攻击 | 高 | 数据窃取 | HTTPS+HSTS |
| 暴力破解 | 中 | 账户接管 | 限流+验证码 |
| 敏感信息泄露 | 中 | 基础设施暴露 | 响应头清理+错误页 |

### 5.2 全栈安全架构

```
┌───────────────────────────────────────────────────────┐
│  第一层：网络层安全                                     │
│  ├── 阿里云安全组：仅开放80/443/22端口                   │
│  ├── DDoS防护：阿里云基础DDoS防护（免费5Gbps）           │
│  └── 入侵检测：fail2ban自动封禁异常IP                    │
├───────────────────────────────────────────────────────┤
│  第二层：传输层安全                                     │
│  ├── SSL/TLS：Let's Encrypt免费证书                     │
│  ├── HSTS：Strict-Transport-Security max-age=31536000   │
│  ├── TLS 1.2+：禁用TLS 1.0/1.1                         │
│  └── HTTP→HTTPS自动跳转                                │
├───────────────────────────────────────────────────────┤
│  第三层：应用层安全                                     │
│  ├── Nginx安全头：                                      │
│  │   ├── X-Frame-Options: DENY                         │
│  │   ├── X-Content-Type-Options: nosniff               │
│  │   ├── X-XSS-Protection: 1; mode=block              │
│  │   ├── Content-Security-Policy: default-src 'self'   │
│  │   └── Referrer-Policy: strict-origin-when-cross-origin│
│  ├── CORS白名单：仅允许自定义域名+localhost              │
│  ├── 限流：Nginx limit_req 10r/s per IP                │
│  └── 请求体大小限制：client_max_body_size 10M           │
├───────────────────────────────────────────────────────┤
│  第四层：API层安全                                      │
│  ├── JWT鉴权：HS256签名，2h过期                         │
│  ├── 接口限流：@RateLimiter注解                         │
│  ├── 参数校验：@Valid + Hibernate Validator             │
│  ├── SQL注入防护：MyBatis-Plus参数化查询                │
│  └── 敏感数据脱敏：日志中隐藏userId/sessionId           │
├───────────────────────────────────────────────────────┤
│  第五层：数据层安全                                     │
│  ├── Redis：bind 127.0.0.1 + requirepass + 禁用FLUSH   │

│  ├── Pulsar：authentication + authorization            │
│  └── 备份：每日mysqldump + RDB持久化                    │
├───────────────────────────────────────────────────────┤
│  第六层：监控与响应                                     │
│  ├── Prometheus告警：服务不可用/高延迟/高错误率          │
│  ├── Grafana看板：安全事件可视化                        │
│  ├── Nginx访问日志：异常请求分析                        │
│  └── fail2ban：自动封禁暴力破解IP                       │
└───────────────────────────────────────────────────────┘
```

### 5.3 Nginx安全配置

```nginx
server {
    listen 443 ssl http2;
    server_name yulike.dev;

    ssl_certificate /etc/letsencrypt/live/yulike.dev/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yulike.dev/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Frame-Options "DENY" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; connect-src 'self' https://api.yulike.dev" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
    limit_req zone=api burst=20 nodelay;

    location / {
        root /var/www/portfolio;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:9199/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /doc.html {
        proxy_pass http://127.0.0.1:9199/doc.html;
    }

    location /grafana/ {
        proxy_pass http://127.0.0.1:3000/;
        proxy_set_header Host $host;
    }

    location /prometheus/ {
        proxy_pass http://127.0.0.1:9090/;
        proxy_set_header Host $host;
        auth_basic "Prometheus";
        auth_basic_user_file /etc/nginx/.htpasswd;
    }
}

server {
    listen 80;
    server_name yulike.dev;
    return 301 https://$server_name$request_uri;
}
```

## 六、自定义域名筛选

### 6.1 域名命名策略

| 策略 | 说明 | 示例 |

### 6.2 候选域名清单

| ⭐2 | **yulike.io** | yu-like + .io科技域名 | ⭐⭐⭐⭐⭐ | 需验证 | ~$35 |
| ⭐3 | **yulike.tech** | yu-like + .tech技术域名 | ⭐⭐⭐⭐ | 需验证 | ~$8 |
| 4 | **yulike.app** | yu-like + .app应用域名 | ⭐⭐⭐⭐ | 需验证 | ~$15 |

| 7 | **yulikepro.com** | yu-like + pro专业 | ⭐⭐⭐ | 需验证 | ~$10 |

### 6.3 域名推荐理由

**首选 `yulike.dev`**：
- `.dev` 是Google推出的开发者专属TLD，强制HTTPS，技术感极强
- 与项目名"yu-like"直接对应，品牌一致性
- HR/面试官看到.dev域名立即联想到开发者作品集
- 年费约$12，性价比高
- Google搜索对.dev域名有开发者内容偏好

**备选 `yulike.io`**：
- `.io` 是科技创业圈最流行的TLD
- 品牌感强，但年费较高(~$35)
- 适合未来扩展为产品官网

### 6.4 域名注册推荐

| 注册商 | .com价格 | .dev价格 | .io价格 | WHOIS隐私 | 推荐 |
|--------|----------|----------|---------|-----------|------|
| Spaceship | $9.08 | ~$12 | ~$35 | 免费 | ✅ 首选 |
| Cloudflare | $10.44 | ~$12 | ~$35 | 免费 | 备选 |
| Porkbun | $11.06 | ~$12 | ~$35 | 免费 | 备选 |

### 6.5 域名验证步骤

1. 访问 [Spaceship](https://www.spaceship.com/) 或 [Porkbun](https://porkbun.com/)
2. 搜索 `yulike.dev` / `yulike.io` / `yulike.tech`
3. 确认可用后立即注册
4. 配置DNS指向阿里云服务器your-server-ip
5. 申请Let's Encrypt SSL证书
