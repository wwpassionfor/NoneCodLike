### 2.5 Redis Hash数据验证（点赞后）

点赞后Redis中的数据结构：

```
thumb:1 (Hash)          ← 用户1的点赞状态
  field: "1" value: "1"  ← 已点赞博客1
  field: "2" value: "1"  ← 已点赞博客2

thumb:2 (Hash)          ← 用户2的点赞状态
  field: "1" value: "1"  ← 已点赞博客1

thumb:3 (Hash)          ← 用户3的点赞状态
  field: "1" value: "1"  ← 已点赞博客1

Redis键数量: 6 (3个Session + 3个thumb Hash)
```

> **验证结论**：Lua脚本HSET原子写入生效，每个用户的点赞状态以Hash结构存储在Redis中。

## 五、数据流转完整性验证

### 5.1 完整数据流图

```
用户1点赞博客1
    ↓ HTTP POST /api/thumb/do
[ThumbController] → [ThumbServiceMQImpl]
    ↓
[Redis Lua脚本] HEXISTS检查 → 未存在
    ↓
[HSET] thumb:1 {1: 1}        ← Redis Hash写入 ✅
    ↓
[Pulsar sendAsync] thumb-topic → ThumbEvent(userId=1, blogId=1, type=INCR)
    ↓ 异步
[ThumbConsumer] 批量消费 → @Transactional
    ↓
[MySQL INSERT] thumb表 (userId=1, blogId=1)
    ↓
[MySQL UPDATE] blog SET thumbCount = thumbCount + CASE ... END
    ↓
[ACK] Pulsar确认消息
```

### 5.2 各层数据一致性

| 层 | 用户1→博客1 | 用户1→博客2 | 用户2→博客1 | 用户3→博客1 |
|----|------------|------------|------------|------------|
| Redis Hash | ✅ 存在 | ✅ 存在 | ✅ 存在 | ❌ 已删除 |
| Pulsar Topic | ✅ 已投递 | ✅ 已投递 | ✅ 已投递 | ✅ DECR已投递 |
| MySQL thumb | ✅ id=...0017 | ✅ id=...338 | ✅ id=...682 | ❌ 无记录 |
| MySQL thumbCount | +1 | +1 | +1 | 无变化 |

**一致性结论**：✅ 四层数据完全一致
